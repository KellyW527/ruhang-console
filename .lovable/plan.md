我会把这次问题当作“保存自评后任何自动推进都不允许”的彻底修复来做。

## 发现的问题
当前弹窗自动关闭的关键原因不是 `SelfEval.onSaved` 直接调用了下一任务，而是 UI 显示条件仍然把“任务体验问卷”绑定在 `feedbackStatus.status === "done"` 上：

```text
保存自评后：任务仍是 feedback_pending
体验问卷：只有 done 才显示
进入下一任务：advance() 会把任务改成 done，然后立刻 setFeedbackTask(null)
```

所以用户保存自评后，要么看不到“这个任务你的体验如何”，要么只有点击“进入下一个任务”后状态变为 done 才短暂出现，但弹窗马上关闭，导致看起来仍然是“自动关闭/自动推进”。

## 修改计划

1. **调整任务体验问卷的显示时机**
   - 把 `TaskFeedbackBar` 从“任务 done 后显示”改成“自评已保存且当前提交通过时显示”。
   - 也就是在 `feedback_pending + selfEvalReady` 阶段就显示“这个任务你的体验如何？”。
   - 这样用户保存自评后，弹窗会停留，并立即看到问卷。

2. **把“进入下一个任务”改成真正的手动动作**
   - 保留底部按钮，但不再让任何保存动作、状态同步、问卷提交自动调用 `advance()`。
   - 只有用户明确点击“进入下一个任务”时，才调用 `finalizeTaskAndUnlock()`，并关闭弹窗。

3. **关闭弹窗的兜底收紧**
   - 在反馈弹窗里阻止点击外部区域或按 Esc 意外关闭，避免保存后的 toast、焦点变化、误触导致弹窗消失。
   - 仍保留右上角关闭按钮/明确关闭按钮，让用户可主动关闭。

4. **清理/加固自动状态同步**
   - 检查 `syncProgressState`、`ensureActiveTask`、`advance`、`openFeedbackForTask` 的相互影响。
   - 确保 `feedback_pending` 不会因为自评保存而被自动改成 `done`。
   - 确保 `current_task_index` 只在手动进入下一任务时变化。

5. **验证**
   - 跑 TypeScript 检查。
   - 重点确认流程：提交任务 → 打开反馈/自评 → 保存自评 → 弹窗保持打开 → 显示任务体验问卷 → 用户手动点击“进入下一个任务”才推进。
